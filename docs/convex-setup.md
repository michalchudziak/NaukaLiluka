# Convex Setup

This app now uses Convex for cloud state instead of Supabase.

## 1. Create the Convex project

1. Open the Convex dashboard: [https://dashboard.convex.dev](https://dashboard.convex.dev)
2. Create a new project.
3. Keep the project open so you can connect this repo to it.

Official guides:

- React Native / Expo quickstart: [https://docs.convex.dev/quickstart/react-native](https://docs.convex.dev/quickstart/react-native)
- Dashboard and project setup: [https://docs.convex.dev/production/hosting/](https://docs.convex.dev/production/hosting/)

## 2. Link this repo to the project

Run:

```bash
npm run convex:dev
```

That command will:

- ask you to sign in if needed
- connect the local repo to a Convex deployment
- create the local Convex project metadata
- enable generated files in `convex/_generated/`

## 3. Add the Expo public URL

Set the deployment URL in your Expo env file:

```bash
EXPO_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud
```

You can copy that URL from the Convex dashboard deployment settings or from the env file generated by `convex dev`.

## 4. Start the app

Run:

```bash
npm start
```

If Expo was already running, restart it after changing env vars.

## 5. Deploy backend changes

When you want to push the Convex backend:

```bash
npm run convex:deploy
```

## Notes

- `npm run convex:codegen` works after the repo has been linked to a Convex deployment.
- The app currently uses the Convex backend modules in `convex/` and reads `EXPO_PUBLIC_CONVEX_URL` at runtime.
